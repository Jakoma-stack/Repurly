# 3.Repurly - launch pack

This pack combines the stronger commercial codebase, public marketing site, launch assets, and sales materials under the **Repurly** brand.

## What is inside
- `app/` - the main application codebase to put in a private GitHub repo
- `public_site/` - the public marketing site for `repurly.org`
- `launch_pack/` - pricing, homepage copy, outreach, sales scripts, and launch docs

## Commercial position
Use **Repurly by Jakoma** as the public brand line. Keep Repurly as the product brand, and use Jakoma as the trust layer.

## Recommended colour direction
- primary blue: `#3B82F6`
- supporting teal: `#14B8A6`
- premium violet: `#8B5CF6`
- deep ink: `#0B1020`

## What still needs real-world validation before charging heavily
No zip can guarantee production readiness. Before taking live client money, test these in your own environment:
1. billing with live Stripe test and webhooks
2. email flows and support inbox
3. live publishing credentials and retries
4. onboarding and team invitation flow
5. legal details, cancellation terms, and privacy wording

## What to do first
1. Put `app/` into a private GitHub repo.
2. Put `public_site/` on `repurly.org` or connect it to the marketing section of your main deployment.
3. Replace placeholder links and emails with your real ones.
4. Use the launch docs to sell a paid pilot before trying to scale self-serve signups.
5. List Repurly on the Jakoma site as a product or app, but keep the main conversion path on `repurly.org`.
