# Repurly final go-live guide

## Recommended structure
- `app/` -> private GitHub repo
- `public_site/` -> marketing site for `repurly.org`
- `launch_pack/` -> pricing, sales, demo, and outbound materials

## Best commercial setup
- Public brand: **Repurly**
- Trust line: **Built by Jakoma**
- Parent site: add Repurly to the Jakoma site as an app/tool with a short product overview and a link through to `repurly.org`
- Primary conversion path: keep demos and signups on `repurly.org`

## Before taking live money
1. Test live Stripe checkout and webhooks.
2. Test email invites, password reset, and support inbox.
3. Test LinkedIn credential handling, retries, and failure recovery.
4. Check onboarding flow from first login to first scheduled post.
5. Replace placeholder domains, support emails, and booking links.
6. Keep the repo private and never upload real secrets.

## Best growth angle
Sell Repurly as a **content operations system** for agencies and founder-led B2B brands. Do not position it as just another AI writer.
