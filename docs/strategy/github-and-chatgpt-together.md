# GitHub and ChatGPT together

This workspace is set up so the same folder can support both:

## In GitHub
Treat this as the main repository. The codebase lives at the root.

## In ChatGPT
Treat this as the project bundle. The most important guidance is in `_chatgpt/` and `docs/decision/`.

## Practical workflow
1. Make code changes in GitHub.
2. Keep planning and execution docs in `docs/`.
3. When you start a fresh chat, upload the current repo zip or the key files plus `_chatgpt/`.
4. Ask ChatGPT to ground recommendations in the actual code and the decision docs.
